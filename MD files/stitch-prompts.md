# Google Stitch Prompts — Marketplace SaaS Screens

> Attach `DESIGN.md` to the Stitch chat before running each prompt below. Run them one at
> a time, in a new prompt each — review and refine each screen before moving to the next,
> so every screen stays consistent with the same design tokens.

---

## Prompt 1 — Storefront Homepage
```
Design the public storefront homepage for a multi-vendor marketplace (like a mini Amazon).
Include:
1. Top bar (logo, search, account icon, cart icon)
2. Hero banner: Primary Indigo background, headline "Shop from independent vendors, all in
one place", a short supporting line, and a "Browse Products" button
3. A row of 5-6 category filter chips (Electronics, Home, Accessories, etc.)
4. "Featured Products" section header + the 4-column product grid
5. A second section "New Arrivals" with the same card style
6. Footer with "marketPlace" link

and layout rules in the attached DESIGN.md exactly.
```

## Prompt 2 — Product Detail Page
```
Design a product detail page for the same marketplace. Layout: large product image on the
left, and on the right: product name, price, vendor name, a short description, a quantity
selector, and an "Add to Cart" button. Keep it simple, two-column, no clutter. Follow the
brand tokens and layout rules in the attached DESIGN.md exactly.
```

## Prompt 3 — Customer Sign-In / Create Account
```
Design a customer sign-in / create-account screen for the marketplace (separate from any
vendor/seller login). Centered card on the page background: email and password fields,
a primary "Sign In" button, and a link to switch to "Create Account". Keep it minimal —
one clear call to action, no distractions. Follow DESIGN.md exactly.
```

## Prompt 4 — Cart Page
```
Design the cart page for a logged-in customer. Show line items grouped by vendor (each
group has a small vendor name header), each item with a thumbnail image, name, quantity
stepper, and subtotal. Below the groups, an order summary card with the total and a
"Checkout" button. Also show what an empty-cart state looks like. Follow DESIGN.md exactly.
```

## Prompt 5 — Vendor Dashboard (Products + Orders)
```
Design two dashboard screens for a vendor (store owner), sharing the same layout: a left
sidebar (logo, nav: Dashboard, Products, Orders, Warehouses, Purchase Orders, Settings, with
a plan badge at the bottom) and a top bar with a page title and a primary action button.
Screen 1 — Products: a data table (SKU, name, price, quantity, status badge) with an
"Add Product" button. Screen 2 — Orders: a data table (order #, customer, total, status
badge) with a status filter. Follow DESIGN.md exactly.
```

## Prompt 6 — Platform Admin Dashboard
```
Design the platform admin dashboard — same sidebar/topbar pattern as the vendor dashboard,
but the nav only shows "Vendors" and "Settings". Main content: a data table of all vendors
(store name, plan badge, active/inactive toggle, product count, order count). Follow
DESIGN.md exactly.
```
